# Viewer MVP 체크리스트

## 문서 목적

이 문서는 현재 단계에서 `web-viewer`를 먼저 완성하기 위해 필요한 MVP 범위를 정리한 문서입니다.

지금 목표는 다음과 같습니다.

- React + Three.js + TypeScript 기반 뷰어를 안정적으로 동작시키기
- 실제로 STL / OBJ를 불러와 화면에서 확인할 수 있게 만들기
- 면접과 포트폴리오에서 설명 가능한 수준의 기능을 우선 완성하기
- WebAssembly 전환 전에 뷰어 UX와 기능 흐름을 먼저 검증하기

## 현재 전략

현재는 최종 구조인 `C++ -> WebAssembly -> web-viewer`로 바로 가지 않고, 먼저 뷰어 기능을 완성하는 단계입니다.

이 단계에서 중요한 점은:

- 사용자가 파일을 올리면 바로 보이는가
- 메쉬를 조작하고 분석할 수 있는가
- 기본적인 의료용 3D Viewer 경험이 갖춰져 있는가

즉, 지금은 `렌더링과 사용자 경험`을 먼저 완성하는 단계입니다.

## 1. 파일 로딩

### 필수

- STL 파일 로드
- OBJ 파일 로드
- ASCII STL 지원
- Binary STL 지원
- 로드 실패 시 에러 메시지 표시
- 파일명 표시

### 완료 기준

- 사용자가 `.stl`, `.obj` 파일을 올리면 에러 없이 로드된다
- 지원하지 않는 형식 또는 잘못된 데이터인 경우 안내 메시지가 나온다

## 2. 기본 뷰어 동작

### 필수

- Three.js scene 초기화
- Perspective camera
- OrbitControls
- 확대 / 축소 / 회전
- 모델 중심 정렬
- 모델 로드 후 카메라 자동 위치 조정

### 완료 기준

- 로드한 모델을 마우스로 자연스럽게 회전, 이동, 확대/축소할 수 있다
- 모델이 너무 멀리 있거나 화면 밖으로 사라지지 않는다

## 3. 렌더링 모드

### 필수

- Solid 렌더링
- Wireframe 토글
- Bounding Box 표시 토글
- DoubleSide 재질 적용 여부 검토
- 기본 조명 세팅

### 완료 기준

- 모델이 기본적으로 잘 보인다
- Wireframe과 Bounding Box를 켜고 끌 수 있다

## 4. 메쉬 정보 패널

### 필수

- 파일명
- 포맷
- Vertex 수
- Triangle 수
- Bounds 표시
- 분석 소스 표시

### 권장

- `TypeScript parser` 결과와 `cpp-core 분석` 결과를 구분해 보여주기

### 완료 기준

- 파일을 로드하면 오른쪽 또는 왼쪽 패널에 메쉬 요약 정보가 즉시 나타난다

## 5. Section Plane

### 필수

- clipping plane 1개 적용
- 슬라이더로 위치 이동
- 모델이 잘리는 것이 눈에 보여야 함

### 권장

- 축 전환 기능
- plane on/off 토글

### 완료 기준

- 사용자가 단면 위치를 움직이며 메쉬 내부를 확인할 수 있다

## 6. Picking

### MVP 포함 권장

- 클릭으로 메쉬 지점 또는 객체 선택
- 선택 위치 하이라이트
- 선택 상태 표시

### 완료 기준

- 사용자가 메쉬를 클릭했을 때 선택 피드백이 눈에 보인다

## 7. FPS / 상태 표시

### 권장

- FPS 표시
- 로딩 중 / 성공 / 실패 상태 메시지
- 현재 분석 소스 표시

### 완료 기준

- 사용자가 현재 뷰어 상태를 쉽게 이해할 수 있다

## 8. UI 정리

### 필수

- 파일 업로드 영역
- 뷰 옵션 영역
- 정보 패널 영역
- 상태 메시지 영역

### 권장

- 모바일 대응
- 여백, 정렬, 라벨 문구 정리
- 버튼 / 토글 시각적 일관성

### 완료 기준

- 처음 보는 사람도 기능 위치를 쉽게 이해할 수 있다

## 9. cpp-core 연동 관련 현재 범위

현재 단계에서 `cpp-core`는 아래까지만 연결되어 있어도 충분합니다.

- STL 메타데이터 분석
- Triangle 수
- Vertex 수
- Bounding Box 계산

이 단계에서는 아직 `cpp-core가 렌더링용 vertex를 직접 넘기지 않아도 괜찮습니다.`

핵심은:

- 뷰어는 완성도 있게 먼저 동작
- `cpp-core`는 분석 역할로 의미를 가짐
- 이후 WebAssembly 전환 시 계산 코어를 자연스럽게 옮길 수 있음

## 10. 지금 단계에서 하지 않아도 되는 것

아래 항목은 지금 당장 MVP를 막을 정도로 우선순위가 높지 않습니다.

- WebAssembly 연결
- BVH 최적화
- 거리 측정
- 다중 section plane
- 고급 material
- shadow
- deferred rendering
- instancing
- culling

## 11. 우선순위 추천

추천 구현 순서는 아래와 같습니다.

1. STL / OBJ 로딩 안정화
2. 카메라 / 자동 프레이밍 안정화
3. Mesh info 패널 정리
4. Wireframe / Bounding Box / Section Plane 정리
5. Picking 추가
6. FPS 및 상태 표시 추가
7. UI 마감

## 12. MVP 완료 기준

다음 조건을 만족하면 viewer MVP로 충분합니다.

- STL / OBJ 파일이 안정적으로 열린다
- 모델이 화면에 정상적으로 렌더링된다
- Orbit 조작이 자연스럽다
- Wireframe, Bounding Box, Section Plane이 동작한다
- Mesh info가 올바르게 보인다
- STL 업로드 시 `cpp-core` 분석 결과가 표시된다
- 포트폴리오 데모로 보여줄 수 있을 만큼 UI가 정리되어 있다
